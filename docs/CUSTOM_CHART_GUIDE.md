# 🗺️ 커스텀 차트 제작 및 설정 해답지 (Custom Chart Guide)

이 문서는 모드 내부의 커스텀 곡 시스템(`HwaResourceManager`, `HwaBattleMediaController`, `BmsParser`)을 사용하여 플레이어가 자신만의 곡, 차트(BMS), 음원(.ogg), 비디오(BGA)를 주입하고 인게임에서 플레이하는 전체 절차와 설정 사양을 정리한 가이드(해답지)입니다.

---

## 📁 1. 전체 폴더 구조 및 파일 구성

게임 설치 디렉터리 내에 `hwa` 폴더를 생성하고, 그 하위에 곡별로 독립된 하위 디렉터리를 구성합니다.

```text
Muse Dash/
├── Mods/
│   └── muse_dash_test.dll          # 빌드 완료된 모드 DLL
└── hwa/                            # 커스텀 리소스 루트 디렉터리
    ├── 01_MyCustomSong/            # 첫 번째 곡 폴더 (알파벳/숫자 순서대로 1999-0에 매핑)
    │   ├── info.txt                # 곡 설정 텍스트 (곡 제목, 난이도 등)
    │   ├── chart.bms               # BMS 차트 파일 (노트 배치 데이터)
    │   ├── music.ogg               # 인게임 재생용 배경음악 (BGM)
    │   └── bga.mp4                 # 인게임 배경 렌더링용 비디오 (BGA - 선택 사항)
    └── 02_SecondSong/              # 두 번째 곡 폴더 (1999-1에 매핑)
        ├── info.txt
        ├── chart.bms
        ├── music.ogg
        └── bga.mp4
```

> [!IMPORTANT]
> - `hwa/` 디렉터리 내부의 하위 폴더들은 **폴더명 알파벳 정렬 순서**에 따라 게임 내의 가상 곡 UID인 `1999-0`, `1999-1`, `1999-2` 순서대로 자동 매핑됩니다.
> - 만약 `hwa/` 폴더 내에 아무런 하위 폴더가 없으면, 모드는 디버그 편의를 위해 **3개의 기본 테스트 슬롯(화영왕 0~2)**을 가상으로 생성합니다.

## 📝 2. 곡 설정 파일 (`info.txt`) 작성 규격

각 곡 폴더 안에는 반드시 메타데이터를 설명하는 텍스트 설정 파일(기본 검색 우선순위: `info.txt` 또는 `info1.txt`)이 존재해야 합니다.

### 설정 항목 목록 및 예시

줄 바꿈을 기준으로 `키: 값` 또는 `키 = 값` 형태로 자유롭게 작성하며, 주석(`//`)을 지원합니다. 대소문자나 공백은 파싱 단계에서 자동으로 정규화됩니다.

```text
// info.txt 예시
가져올곡: music_package_0      // 얇은 복제(Thin Clone)를 수행할 원본 곡 UID (기본 곡: Default Music 등)
가져올앨범: ALBUM1             // 원본 곡이 속한 앨범의 JSON 리소스명
씬번호: 4                     // 플레이할 인게임 배경 테마 번호 (1: 디스코, 2: 우주, 4: 복고풍 등)

커스텀곡제목: 나의 멋진 곡     // UI에 최종 노출될 실제 곡의 한글/다국어 제목
커스텀아티스트: 화영왕        // UI에 최종 노출될 작곡가명
레벨디자이너: 화영왕          // UI에 노출될 채보 제작자 이름

// 난이도 설정 (1: Easy, 2: Hard, 3: Master, 4: Another, 5: Hidden)
난이도1: 4
난이도2: 7
난이도3: 11
```

### 🔍 매개변수 상세 설명

| 한국어 키 후보 | 영어 키 후보 | 타입 | 설명 |
| :--- | :--- | :--- | :--- |
| **곡이름, 곡명, 가져올곡** | `song`, `title`, `music` | 문자열 | 원본 데이터베이스에서 복제해 올 대상 곡의 UID (예: `music_package_0`) |
| **가져올앨범, 앨범** | `album` | 문자열 | 원본 곡이 포함된 앨범 명칭 (예: `ALBUM1`) |
| **씬번호** | `scene` | 정수 | 플레이될 인게임 비주얼 씬 번호 (배경, 장애물 디자인 결정) |
| **커스텀곡제목** | `customtitle` | 문자열 | 인게임 UI(곡 선택, 준비 화면)에 표시할 실제 한글 제목 |
| **커스텀아티스트** | `customartist` | 문자열 | 인게임 UI에 표시할 아티스트 이름 |
| **레벨디자이너** | `leveldesigner` | 문자열 | 인게임 UI에 표시할 채보 디자이너 명칭 |
| **난이도1 ~ 난이도5** | `difficulty1 ~ 5` | 정수 | 곡 선택 화면에 표시될 난이도 레벨 수치 값 |

## 📊 3. BMS 차트 파일 (`.bms`) 작성 및 매핑

모드는 곡 폴더 내에서 파일명이 `chart.bms`, `main.bms`, `test.bms`인 것을 최우선으로 탐색하며, 없을 경우 첫 번째 발견된 `.bms` 파일을 파싱합니다.

BMS 내의 `#WAVxx` 리소스 선언 규칙과 노트 타입 매핑의 핵심 요약은 다음과 같습니다 (상세 매핑은 [BMS 파싱 및 매핑 가이드](file:///h:/source/repos/muse%20dash%20test/docs/BMS_PARSING.md) 참조).

### WAV 파일명과 인게임 노트의 핵심 관계
BMS 차트 프로그램(BMSE 등)에서 `#WAVxx` 영역에 지정할 파일명은 다음과 같은 명명법을 지킵니다:
- **`011001_일반 지상 노멀_dt1.48.wav`**처럼 `[6자리UID]_[디자이너설명]_dt[시간오프셋].wav` 형태를 띱니다.
- **6자리 UID (`zzxxyy` 구조)**:
  - 앞 2자리(`zz`): 씬 계열을 의미합니다. (예: `05` ➡️ `scene_05`)
  - 중간 2자리(`xx`): **노트 타입**을 결정하는 핵심 코드입니다.
    - `10`/`11`/`12`: 일반 단타 노트
    - `02`: 롱노트 (롱노트는 시작 지점과 끝 지점에 동일한 WAV 코드가 쌍으로 매칭되어야 함)
    - `03`: 톱니 장애물
    - `04`: 샌드백 (다중 타격 가능)
    - `06`/`07`/`08`: 보스 발사체 / 투사체
    - `09`: 보스 톱니 장애물
    - `17`: 고스트(유령) 노트
  - 뒤 2자리(`yy`): 레인과 방향을 가리킵니다.
    - `01` 지상 기본, `04` 공중 기본, `07` 지상 Up, `10` 공중 Up, `13` 지상 Down, `16` 공중 Down
- **특수 프리팹 (하트, 음표)**:
  - `0002` 접두사로 시작하면 체력 회복 하트 노트가 되며 효과음이 `sfx_hp`로 보정됩니다.
  - `0003` 접두사로 시작하면 스코어 음표 노트가 되며 효과음이 `sfx_score`로 보정됩니다.

## 🎵 4. 오디오 (BGM) 및 비디오 (BGA) 주입 스펙

### ① 커스텀 BGM (`.ogg`)
- **형식**: 반드시 **OGG Vorbis** 포맷이어야 합니다.
- **파일명 탐색 순서**:
  1. `info.txt` 파일과 확장자를 제외한 파일명이 일치하는 OGG 파일.
  2. 파일명에 `bgm`, `battle`, `music`, `song` 키워드가 들어간 OGG 파일.
  3. 폴더 내 알파벳 기준 첫 번째 OGG 파일.
- **동작**: 유니티의 `UnityWebRequestMultimedia`를 통해 인메모리로 고속 비동기 디코딩된 뒤, 전용 배틀 AudioSource인 `"HwaBattleBgmSource"`를 통해 무한 루프로 안전 재생됩니다.

### ② 커스텀 BGA 비디오 (`.mp4`)
- **형식**: **MP4 (H.264 인코딩)** 비디오 포맷을 권장합니다.
- **파일명 탐색 순서**: 폴더 내 알파벳 기준 첫 번째 MP4 파일.
- **동작**:
  - 인게임 카메라 하위에 `"VideoBackgroundQuad"`라는 평면 메쉬를 동적 빌드합니다.
  - 카메라의 종횡비(`aspect`) 및 화각 크기(`orthographicSize`)를 실시간 계산하여 화면을 꽉 채우는 크기로 Quad 크기를 조정합니다.
  - `Unlit/Texture` 최적화 셰이더 머티리얼을 씌워 `VideoPlayer` 컴포넌트의 Material Override 기술로 오버레이 재생을 진행합니다.
  - 소팅 레이어는 `Background`, 순서는 `-17`로 배치되어 게임 내 인터페이스나 전경 노트 플레이를 방해하지 않고 뒷배경으로 자연스럽게 노출됩니다.

> [!TIP]
> **싱크 정밀도 및 예외 복구**:
> 게임 일시정지(Pause)나 재개(Resume) 시, 모드는 비디오 플레이어(`VideoPlayer`)와 커스텀 오디오(`AudioSource`)를 실시간 동반 제어하여 싱크 어긋남 현상을 원천 방지합니다.

## 🎮 5. 인게임 플레이 순서 (해답)

모든 리소스를 세팅했다면 아래 단계로 플레이합니다.

1. **빌드**: `build.bat`를 실행해 모드를 컴파일하고 Muse Dash의 `Mods/` 디렉터리에 복사합니다.
2. **게임 실행**: 게임을 구동하고 곡 선택 탭으로 들어갑니다.
3. **태그 선택**: 왼쪽 카테고리 태그 중, 모드가 리소스를 가로채 자체 패킹된 파일로 아이콘을 교체한 **"실험 모드 (Experiment Mod)"** 태그를 선택합니다.
4. **앨범 선택**: 실험 모드 내부의 가상 앨범인 **"실험 앨범 (UID: 1998-0)"**을 클릭합니다.
5. **곡 선택 및 플레이**:
   - `hwa/` 폴더에 세팅한 개수대로 곡들이 `1999-0`, `1999-1` 형태로 주입되어 있습니다.
   - 플레이를 원하는 곡을 누르고 들어가 재생 버튼을 누릅니다.
   - 플레이 시작 즉시 BGM이 로드되고 뒷배경에 아름답게 비디오 BGA가 렌더링되며, BMS 파일로 분석된 커스텀 차트의 생생한 노트 플레이를 즐기실 수 있습니다!
